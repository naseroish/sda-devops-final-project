# Contributing Guidelines

Thank you for contributing to the project! Please follow these guidelines to keep our workflow consistent and clean.

## Branching Strategy
- Create a new branch for each feature or fix.
- Use the naming convention:  
  `feature/<name>` for new features  
  `fix/<name>` for bug fixes  
  `docs/<name>` for documentation updates  

## Commit Messages
- Follow the conventional commit format:  
  `type(scope): short description`  
  Example: `feat(api): add expense creation endpoint`

## Pull Requests
1. Ensure your branch is up to date with `main`.
2. Test your changes locally before pushing.
3. Submit a PR with a clear title and description.
4. Assign reviewers from the team.

## Code Standards
- Use TypeScript for all backend and frontend files.
- Ensure linting passes before committing.
- Follow the existing folder structure.

## Documentation
- Update `/docs` when making relevant structural or process changes.

---

Following these rules helps keep our codebase consistent and maintainable.
